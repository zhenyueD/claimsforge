Differentiator #1 · Supervisor

AWS-IAM-style Supervisor —
explicit Deny / Exempt / Cap in pure Python

🚫 L1 · DENY — force_escalate · short-circuit
RULE-FRAUD-REPLAY (pHash) RULE-MULTIMODAL-MISMATCH RULE-DUPLICATE-ORDER RULE-SESSION-FREQ HR-LIQUID-ELECTRONICS · Tier-2 HR-LEGAL-THREAT · Tier-2 HR-LUXURY · Tier-2
↩️ L2 · EXEMPT — un-escalate prior block
RULE-PERISHABLE-EXEMPT — food / cosmetic claims skip the no-image rule
📐 L3 · CAP — clamp without escalating
RULE-MAX-CASH-PAYMENT · $500 RULE-MAX-PCT-ORDER · 100%

Why IAM, not LLM judgement?

LLMs hallucinate policies and overrule cash caps when the customer writing is empathetic. A pure-Python gate cannot be talked out of a hard rule. Every decision logs the rule_id that fired.

3-Tier Rule Authority

Tier 1 — code constitution, can't toggle at runtime
Tier 2 — data/hard_rules.json, business toggle via admin UI
Tier 3 — KB embeddings, soft guidance only
