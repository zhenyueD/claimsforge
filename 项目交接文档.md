## 二、核心组件

### 2.1 四大能力域（agents/）

| 模块 | 文件 | 职责 |
|------|------|------|
| **对话引擎** | `agents/conversation_engine.py` | 意图识别、情绪分析、知识库匹配、需求挖掘 |
| **升级引擎** | `agents/escalation_engine.py` | 工单创建、赔偿决策(<50自动,>=50需审批)、情绪危机处理 |
| **知识管道** | `agents/knowledge_pipeline.py` | 知识库CRUD、命中率统计、缺口检测 |
| **观测者** | `agents/observer.py` | 健康检查、日报生成、晨报简报、满意度统计 |

### 2.2 API 服务（api/main.py）

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/chat` | POST | 核心对话接口 |

| `/api/kb/stats` | GET | 知识库统计 |
| `/api/kb/gaps` | GET | 知识缺口报告 |
| `/api/kb/add` | POST | 添加知识条目 |
| `/api/health` | GET | 系统健康度 |
| `/api/stats` | GET | 运行统计 |
| `/api/report/daily` | POST | 触发日报生成 |
| `/api/report/morning-brief` | GET | 获取晨报简报 |
| `/api/report/list` | GET | 报告列表 |
| `/api/tickets` | GET | 工单列表 |
| `/api/history/{session_id}` | GET | 会话历史 |
| `/api/notify/queue` | GET | 待处理通知队列 |
| `/api/notify/process` | POST | 标记通知已处理 |
| `/ws` | WebSocket | 实时事件推送 |
| `/` | GET | 前端页面 |

### 2.3 数据文件（data/）

- `knowledge_base.json` — 知识库条目（含8条虚拟产品数据）
- `state.json` — 运行状态与统计
- `tickets.json` — 工单记录
- `satisfaction.json` — 满意度评分
- `email_queue.json` — 邮件队列
- `notify_queue.json` — 微信通知队列

## 三、关键业务规则

### 3.1 赔偿决策
- **<50元**: AI自主批准，直接回复用户
- **>=50元**: 挂起待审批，微信通知用户（当前为队列化，需外部触发）

### 3.2 情绪危机
- 情绪评分 ≤2.0: 紧急微信通知 + 组长接管
- 情绪评分 ≤5.0: 升级工单 + 组长介入

### 3.3 报告机制
- **日报**: 每日20:00生成，发送至 23992721@qq.com（当前为队列模式）
- **晨报**: 每日08:00生成，包含今日预警 + 待处理事项
